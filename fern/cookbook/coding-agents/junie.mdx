## Using Junie CLI with kenh-ai

[Junie CLI](https://junie.jetbrains.com/) is an agentic coding tool by JetBrains that provides an interactive terminal interface for developers to review, write, and modify code. By connecting Junie to kenh-ai, you can access hundreds of AI models from a single API key instead of managing separate keys for each provider.

### Why Use kenh-ai with Junie?

- **Access to hundreds of models** -- Use models from Anthropic, OpenAI, Google, xAI, Meta, and many more through a single API key
- **Provider failover** -- If one provider is unavailable or rate-limited, kenh-ai automatically routes to another
- **Centralized billing** -- Track and manage spending across all models from your [kenh-ai dashboard](https://kenh-ai.com/activity)
- **Team controls** -- Set budgets and monitor usage across your organization

### Prerequisites

- [Junie CLI](https://junie.jetbrains.com/) installed (see [Step 1](#step-1-install-junie-cli) below)
- An [kenh-ai API key](https://kenh-ai.com/settings/keys)

### Step 1: Install Junie CLI

<Tabs>
  <Tab title="Linux / macOS">
    ```bash lines
    curl -fsSL https://junie.jetbrains.com/install.sh | bash
    ```
  </Tab>

  <Tab title="Windows">
    ```powershell lines
    powershell -NoProfile -ExecutionPolicy Bypass -Command "iex (irm 'https://junie.jetbrains.com/install.ps1')"
    ```
  </Tab>

  <Tab title="Homebrew">
    ```bash lines
    brew tap jetbrains-junie/junie
    brew update
    brew install junie
    ```
  </Tab>
</Tabs>

Verify the installation:

```bash lines
junie --version
```

### Step 2: Configure kenh-ai

Junie supports kenh-ai as a built-in BYOK (Bring Your Own Key) provider. Set the `JUNIE_KENH_AI_API_KEY` environment variable to connect Junie to kenh-ai:

<Tabs>
  <Tab title="Shell Profile (Recommended)">
    Add the environment variable to your shell profile for persistent configuration:

    ```bash lines
    # Open your shell profile
    nano ~/.zshrc  # or ~/.bashrc for Bash users

    # Add this line:
    export JUNIE_KENH_AI_API_KEY="<your-openrouter-api-key>"
    ```

    After saving, restart your terminal or run `source ~/.zshrc` for changes to take effect.
  </Tab>

  <Tab title="Inline">
    You can also pass the API key directly when running Junie:

    ```bash lines
    junie --openrouter-api-key "<your-openrouter-api-key>"
    ```
  </Tab>

  <Tab title="CI/CD">
    In CI/CD pipelines, set the environment variable in your runner configuration:

    ```bash lines
    export JUNIE_KENH_AI_API_KEY="<your-openrouter-api-key>"
    junie "Review and fix any code quality issues in the latest commit"
    ```
  </Tab>
</Tabs>

<Note>
Replace `<your-openrouter-api-key>` with your actual kenh-ai API key from the [API Keys page](https://kenh-ai.com/settings/keys). Keys start with `sk-or-v1-`.
</Note>

### Step 3: Start Coding

Navigate to your project directory and start Junie:

```bash lines
cd /path/to/your/project
junie
```

Type your prompt in the interactive CLI:

```text lines
> give me an overview of this codebase
```

Use `@` to attach files or folders to the request context, and type `/` to see available slash commands.

### Headless Mode (CI/CD)

Junie supports a headless mode for non-interactive use in CI/CD pipelines. Combined with kenh-ai, this gives you centralized billing and model access for automated coding tasks:

```bash lines
# Install Junie CLI
curl -fsSL https://junie.jetbrains.com/install.sh | bash

# Run Junie with kenh-ai in headless mode
export JUNIE_KENH_AI_API_KEY="$KENH_AI_API_KEY"
junie "Review and fix any code quality issues in the latest commit"
```

#### GitHub Actions Example

```yaml lines
name: Code Review
on:
  pull_request:
    types: [opened, synchronize]
jobs:
  review:
    runs-on: ubuntu-latest
    permissions:
      contents: read
      pull-requests: write
      issues: write
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 1
      - uses: JetBrains/junie-github-action@v0
        with:
          openrouter_api_key: ${{ secrets.KENH_AI_API_KEY }}
          prompt: "code-review"
```

#### GitLab CI/CD Example

Follow the
[setup instructions](https://junie.jetbrains.com/docs/junie-gitlab-ci-cd.html)
to configure the Junie Workspace project, then add
`JUNIE_KENH_AI_API_KEY` as a CI/CD variable. Once
set up, trigger reviews by commenting on any MR:

```text lines
#junie code-review
```

### Verify Your Connection

After starting a session, check the [kenh-ai Activity Dashboard](https://kenh-ai.com/activity) to confirm your requests are being routed through kenh-ai.
