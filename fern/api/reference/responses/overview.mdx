<Warning>
**Beta API**

  This API is in **beta stage** and may have breaking changes. Use with caution in production environments.
</Warning>

<Info>
**Stateless Only**

  This API is **stateless** - each request is independent and no conversation state is persisted between requests. You must include the full conversation history in each request.
</Info>

kenh-ai's Responses API Beta provides OpenAI-compatible access to multiple AI models through a unified interface, designed to be a drop-in replacement for OpenAI's Responses API. This stateless API offers enhanced capabilities including reasoning, tool calling, and web search integration, with each request being independent and no server-side state persisted.

## Base URL

```lines
https://kenh-ai.com/api/v1/responses
```

## Authentication

All requests require authentication using your kenh-ai API key:

<CodeGroup>

```typescript title="TypeScript" lines
const response = await fetch('https://kenh-ai.com/api/v1/responses', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_KENH_AI_API_KEY',
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    model: 'openai/o4-mini',
    input: 'Hello, world!',
  }),
});
```

```python title="Python" lines
import requests

response = requests.post(
    'https://kenh-ai.com/api/v1/responses',
    headers={
        'Authorization': 'Bearer YOUR_KENH_AI_API_KEY',
        'Content-Type': 'application/json',
    },
    json={
        'model': 'openai/o4-mini',
        'input': 'Hello, world!',
    }
)
```

```bash title="cURL" lines
curl -X POST https://kenh-ai.com/api/v1/responses \
  -H "Authorization: Bearer YOUR_KENH_AI_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "openai/o4-mini",
    "input": "Hello, world!"
  }'
```

</CodeGroup>

## Error Handling

The API returns structured error responses:

```json lines
{
  "error": {
    "code": "invalid_prompt",
    "message": "Missing required parameter: 'model'."
  },
  "metadata": null
}
```

For comprehensive error handling guidance, see [Error Handling](./error-handling).
