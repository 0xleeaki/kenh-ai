The agent toolkit (`callModel`, `tool`, stop conditions, etc.)
has moved from `@kenh-ai/sdk` to a standalone
**`@kenh-ai/agent`** package. The agent package includes
its own `kenh-ai` client class, so you no longer need
`@kenh-ai/sdk` as a dependency for agent workflows.

## Who needs to migrate?

You need to migrate if your code imports any of the following
from `@kenh-ai/sdk`:

- `callModel` / `ModelResult`
- `tool` / `Tool` / tool type guards
- Stop conditions (`stepCountIs`, `hasToolCall`, etc.)
- Async parameters (`CallModelInput`, `resolveAsyncFunctions`)
- Conversation state helpers
- Message format converters (`fromClaudeMessages`,
  `fromChatMessages`, etc.)

If you only use the REST API client for non-agent features
(`client.chat.send(...)`, `client.models.list()`, etc.),
**no changes are needed**.

## Step 2: Update imports

Replace `@kenh-ai/sdk` subpath imports with
the equivalent `@kenh-ai/agent` subpath.

### Client class

`@kenh-ai/agent` ships its own `kenh-ai` client, so
you can drop the `@kenh-ai/sdk` dependency entirely if
you only use agent features:

```diff lines
- import KenhAI from '@kenh-ai/sdk';
- import { callModel } from '@kenh-ai/sdk/funcs/call-model';
+ import { KenhAI } from '@kenh-ai/agent';

  const client = new KenhAI({
    apiKey: process.env.KENH_AI_API_KEY,
  });
+
+ const result = client.callModel({
+   model: 'openai/gpt-4o',
+   input: 'Hello',
+ });
+ const text = await result.getText();
```

You can also import the client from a direct subpath:

```typescript lines
import { KenhAI } from '@kenh-ai/agent/client';
```

### Core imports

```diff lines
- import { callModel } from '@kenh-ai/sdk/funcs/call-model';
+ import { callModel } from '@kenh-ai/agent/call-model';

- import { ModelResult } from '@kenh-ai/sdk/lib/model-result';
+ import { ModelResult } from '@kenh-ai/agent/model-result';

- import { tool } from '@kenh-ai/sdk/lib/tool';
+ import { tool } from '@kenh-ai/agent/tool';
```

### Tool types and guards

```diff lines
- import type { Tool } from '@kenh-ai/sdk/lib/tool-types';
+ import type { Tool } from '@kenh-ai/agent/tool-types';

- import {
-   hasExecuteFunction,
-   isGeneratorTool,
- } from '@kenh-ai/sdk/lib/tool-types';
+ import {
+   hasExecuteFunction,
+   isGeneratorTool,
+ } from '@kenh-ai/agent/tool-types';
```

### Stop conditions

```diff lines
- import {
-   stepCountIs,
-   hasToolCall,
-   maxCost,
- } from '@kenh-ai/sdk/lib/stop-conditions';
+ import {
+   stepCountIs,
+   hasToolCall,
+   maxCost,
+ } from '@kenh-ai/agent/stop-conditions';
```

### Async parameters

```diff lines
- import type {
-   CallModelInput,
- } from '@kenh-ai/sdk/lib/async-params';
+ import type {
+   CallModelInput,
+ } from '@kenh-ai/agent/async-params';
```

### Conversation state and message formats

Conversation helpers and message format converters are
available from the package barrel:

```diff lines
- import {
-   createInitialState,
-   updateState,
-   fromClaudeMessages,
-   fromChatMessages,
- } from '@kenh-ai/sdk';
+ import {
+   createInitialState,
+   updateState,
+   fromClaudeMessages,
+   fromChatMessages,
+ } from '@kenh-ai/agent';
```

|---|---|
| `@kenh-ai/sdk` (client class) | `@kenh-ai/agent` or `@kenh-ai/agent/client` |
| `@kenh-ai/sdk/funcs/call-model` | `@kenh-ai/agent/call-model` |
| `@kenh-ai/sdk/lib/model-result` | `@kenh-ai/agent/model-result` |
| `@kenh-ai/sdk/lib/tool` | `@kenh-ai/agent/tool` |

## Automated migration

The script below handles **subpath imports** automatically.
Barrel imports (`from '@kenh-ai/sdk'`) and client class
imports (`import KenhAI from '@kenh-ai/sdk'`) must
be updated **manually** — a blanket replacement on the bare
package name would also match subpath imports and break
your code. See the [Client class](#client-class) and
[Conversation state](#conversation-state-and-message-formats)
sections above for the correct replacements.

```bash lines
# Using sed (macOS)
find src -name '*.ts' -o -name '*.tsx' | xargs sed -i '' \
  -e "s|@kenh-ai/sdk/funcs/call-model|@kenh-ai/agent/call-model|g" \
  -e "s|@kenh-ai/sdk/lib/model-result|@kenh-ai/agent/model-result|g" \
  -e "s|@kenh-ai/sdk/lib/tool-types|@kenh-ai/agent/tool-types|g" \
  -e "s|@kenh-ai/sdk/lib/tool|@kenh-ai/agent/tool|g" \
  -e "s|@kenh-ai/sdk/lib/stop-conditions|@kenh-ai/agent/stop-conditions|g" \
  -e "s|@kenh-ai/sdk/lib/async-params|@kenh-ai/agent/async-params|g"
```

```bash lines
# Using sed (Linux)
find src -name '*.ts' -o -name '*.tsx' | xargs sed -i \
  -e "s|@kenh-ai/sdk/funcs/call-model|@kenh-ai/agent/call-model|g" \
  -e "s|@kenh-ai/sdk/lib/model-result|@kenh-ai/agent/model-result|g" \
  -e "s|@kenh-ai/sdk/lib/tool-types|@kenh-ai/agent/tool-types|g" \
  -e "s|@kenh-ai/sdk/lib/tool|@kenh-ai/agent/tool|g" \
  -e "s|@kenh-ai/sdk/lib/stop-conditions|@kenh-ai/agent/stop-conditions|g" \
  -e "s|@kenh-ai/sdk/lib/async-params|@kenh-ai/agent/async-params|g"
```

<Note>
The `tool-types` replacement runs before `tool` to avoid
partial matches. After running the script, search your
codebase for any remaining `from '@kenh-ai/sdk'` (without
a `/` subpath) to find barrel and client imports that need
manual updates.
</Note>

## FAQ

### Do I still need `@kenh-ai/sdk`?

Only if you use non-agent REST API features like
`client.models.list()`, `client.credits.get()`, or
`client.chat.send()`. If your code only uses `callModel`,
tools, and the agent client, you can remove
`@kenh-ai/sdk` entirely.

### Can I use both packages together?

Yes. They are designed to work side by side. Use
`@kenh-ai/sdk` for REST API features and
`@kenh-ai/agent` for the agent toolkit:

```typescript lines
import { KenhAI } from '@kenh-ai/sdk';
import { callModel } from '@kenh-ai/agent/call-model';
import { tool } from '@kenh-ai/agent/tool';
```

### Will the old imports keep working?

The agent exports will be removed from `@kenh-ai/sdk`
in a future major version. Update your imports now to
avoid a breaking change later.

### Do I need to change my API key or configuration?

No. `@kenh-ai/agent` uses the same API key and
endpoints. No server-side changes are required.
