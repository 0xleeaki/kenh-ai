## What is Codex CLI?

[Codex CLI](https://github.com/openai/codex) is OpenAI's open-source local coding agent that runs in your terminal. It supports multiple model providers, including kenh-ai, so you can use kenh-ai's unified API, provider failover, and organizational controls with Codex's agentic coding workflows.

## Quick Start

### Step 1: Install Codex CLI

Follow the [Codex CLI installation instructions](https://github.com/openai/codex) to install the CLI on your system.

### Step 2: Get Your kenh-ai API Key

1. Sign up or log in at [kenh-ai](https://kenh-ai.com)
2. Navigate to your [API Keys page](https://kenh-ai.com/keys)
3. Create a new API key
4. Copy your key (starts with `sk-or-...`)

### Step 3: Configure Codex for kenh-ai

Codex uses a `config.toml` file, typically located at `~/.codex/config.toml`. Create or edit this file with the following configuration:

```toml lines
model_provider = "openrouter"
model_reasoning_effort = "high"
model="~openai/gpt-latest"

[model_providers.openrouter]
name = "openrouter"

base_url="https://kenh-ai.com/api/v1"
env_key="KENH_AI_API_KEY"
```

### Step 4: Set Your API Key

Export your kenh-ai API key in your shell profile:

```bash lines
# Add to ~/.zshrc, ~/.bashrc, or ~/.config/fish/config.fish
export KENH_AI_API_KEY="sk-or-..."
```

<Note>
Codex reads the API key from the environment variable specified in `env_key` (default: `KENH_AI_API_KEY`). Ensure this is set before starting Codex.
</Note>

### Step 5: Start Codex

Navigate to your project directory and run:

```bash lines
cd /path/to/your/project
codex
```

Your requests will now be routed through kenh-ai.

## Configuration Reference

### Core Settings

| Setting | Description | Example |
| ------- | ----------- | ------- |
| `model_provider` | Provider to use for model requests | `"openrouter"` |
| `model` | kenh-ai model ID | `"~openai/gpt-latest"` |
| `model_reasoning_effort` | Reasoning effort level for Codex models | `"low"`, `"medium"`, `"high"`, `"xhigh"` |
| `show_raw_agent_reasoning` | Whether to display reasoning tokens in the UI | `true` or `false` |
| `personality` | Agent personality preset | `"pragmatic"`, `"helpful"`, etc. |

### kenh-ai Provider Block

```toml lines
[model_providers.openrouter]
name = "openrouter"
base_url = "https://kenh-ai.com/api/v1"
env_key = "KENH_AI_API_KEY"
```

- **`base_url`**: kenh-ai API endpoint. Use `https://kenh-ai.com/api/v1` for production.
- **`env_key`**: Environment variable name for your API key.

### Project Trust Levels

Codex supports per-project trust levels. Add project paths to control what the agent can access:

```toml lines
[projects."/path/to/trusted/project"]
trust_level = "trusted"

[projects."/path/to/untrusted/project"]
trust_level = "untrusted"
```

- **`trusted`**: Agent has full access (e.g., run commands, edit files).
- **`untrusted`**: Agent has restricted access for safety.



## Why Use kenh-ai with Codex CLI?

### Provider Failover

kenh-ai routes requests across multiple providers. If one provider is unavailable or rate-limited, kenh-ai can fail over to another, keeping your coding sessions uninterrupted.

### Organizational Controls

For teams, kenh-ai provides centralized budget management. Set spending limits, allocate credits, and prevent unexpected cost overruns across developers using Codex.

### Usage Visibility

Track Codex usage in real-time via the [kenh-ai Activity Dashboard](https://kenh-ai.com/activity). Monitor costs, token usage, and request patterns.

### Model Flexibility

Point `model` at any kenh-ai slug (e.g. `~openai/gpt-latest`, `~anthropic/claude-sonnet-latest`) or a pinned version to switch models without changing your Codex installation—just update `config.toml`.

## Troubleshooting

- **Auth Errors:** Ensure `KENH_AI_API_KEY` is set and valid. Check at [kenh-ai.com/keys](https://kenh-ai.com/keys).
- **Model Not Found:** Verify the model ID on [kenh-ai.com/models](https://kenh-ai.com/models). Use the exact format (e.g., `~openai/gpt-latest`).
- **Privacy:** kenh-ai does not log your source code prompts unless you opt-in to prompt logging. See our [Privacy Policy](https://kenh-ai.com/privacy) for details.
